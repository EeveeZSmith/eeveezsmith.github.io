HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

Non-semantic tags: tags that have no meaning in ordinary language. For example, the tag "div" gives no indication to a non-programmer as to its purpose.

Semantic tags: tags whose names convey a clear meaning, even to people who are not programmers. For example, "heading" and "image."

What are two (or more) benefits of using semantic tags?

Easy reading: Because they look like familiar terms, rather than gibberish, semantic tags are much easier to read. Not only does this aid newcomers in learning to code, but it makes reviewing, editing, and debugging code easier for professionals.

Accessibility: Semantic tags improve user experience for those using screen readers and other assistive devices.
